LISTE DES PARTIES UI FRONTEND MANQUANTES OU À COMPLÉTER
========================================================
Date : Juin 2026
Projet : DotnetNiger — Community Platform (Blazor WebAssembly)

─────────────────────────────────────────────────────────
1. PAGE DE DÉTAIL D'UNE RESSOURCE (/ressource/{slug})
─────────────────────────────────────────────────────────
   - Aucune page de détail n'existe pour afficher une ressource individuelle
   - Besoin : URL, description complète, niveau, type, tags cliquables,
     compteur de vues, date de publication, auteur/soumis par
   - Backend prêt : ResourcesController.GetById(id), IncrementViewCount

─────────────────────────────────────────────────────────
2. NEWSLETTER — INTERFACE COMPLÈTE
─────────────────────────────────────────────────────────
   - Le footer a un formulaire HTML statique sans logique
   - Aucun service INewsletterService défini dans le frontend
   - Besoin :
     • INewsletterService (interface + implémentations Mock/Api)
     • Enregistrement DI dans Program.cs
     • API subscribe/unsubscribe avec gestion d'état (succès/erreur/déjà abonné)
   - Backend prêt : NewsletterController (subscribe, unsubscribe, count)
   - Gateway prête : routes /api/newsletters/*

─────────────────────────────────────────────────────────
3. FIL D'ACTUALITÉ / ACTIVITÉS RÉCENTES
─────────────────────────────────────────────────────────
   - Page d'accueil (Home.razor) n'affiche pas de flux d'activités
   - Besoin : section "Dernières activités" mélangeant posts, événements, 
     ressources (via l'API /api/v1/search ou un endpoint dédié)
   - Backend : SearchController.SearchAsync() peut servir si type non filtré

─────────────────────────────────────────────────────────
4. PROFIL UTILISATEUR PUBLIC (/membre/{id} ou /profil/{slug})
─────────────────────────────────────────────────────────
   - Aucune page de profil public pour voir un membre
   - Besoin : nom, avatar, bio, réseaux sociaux, activités récentes,
     ressources partagées
   - Backend partiel : ProfileController.GetProfile() (Identity), 
     MemberDirectoryController pourrait exister

─────────────────────────────────────────────────────────
5. PAGE DE RECHERCHE GLOBALE (/recherche)
─────────────────────────────────────────────────────────
   - La recherche existe dans ApiPostService/ApiResourceService 
     via /api/v1/search mais sans UI dédiée
   - Besoin : barre de recherche accessible depuis le header, 
     page de résultats avec onglets (Posts, Événements, Ressources),
     filtres par type, pagination
   - Backend prêt : SearchController.SearchAsync()
   - Gateway prête : /api/v1/search/{everything}

─────────────────────────────────────────────────────────
6. GALLERIE / MÉDIATHÈQUE
─────────────────────────────────────────────────────────
   - Aucune section médias (photos d'événements, captures d'écran)
   - Besoin : upload d'images, galerie par événement, lazy loading

─────────────────────────────────────────────────────────
7. PAGE DE PROJETS COMMUNAUTAIRES (/projets)
─────────────────────────────────────────────────────────
   - Backend a ProjectController avec CRUD complet
   - Gateway : routes /api/projects/{everything}
   - Aucune UI frontend pour lister/créer des projets
   - Besoin : page publique + admin CRUD

─────────────────────────────────────────────────────────
8. PAGE DES MEMBRES (/membres)
─────────────────────────────────────────────────────────
   - Backend : MemberDirectoryController (à vérifier)
   - Gateway : /api/members/{everything}
   - Aucune UI frontend pour le répertoire des membres

─────────────────────────────────────────────────────────
9. PARTENAIRES (/partenaires)
─────────────────────────────────────────────────────────
   - Backend : PartnerController (à vérifier)
   - Gateway : /api/partners/{everything}
   - Aucune UI frontend pour afficher les partenaires

─────────────────────────────────────────────────────────
10. GESTION DES TAGS / CATÉGORIES (Admin)
─────────────────────────────────────────────────────────
   - Le formulaire ressource admin (Ressource.razor) n'a pas 
     d'input pour les tags (TagNames) ni les catégories (CategoryIds)
   - Backend les supporte mais le frontend ne les expose pas
   - Besoin : sélecteur de tags, sélecteur de catégories dans 
     le drawer d'édition

─────────────────────────────────────────────────────────
11. CONFIRMATION DE SUPPRESSION
─────────────────────────────────────────────────────────
   - Admin/Ressource.razor : DeleteResource() supprime sans 
     boîte de confirmation
   - Besoin : modal de confirmation "Êtes-vous sûr ?" avant suppression

─────────────────────────────────────────────────────────
12. NOTIFICATIONS / TOASTS
─────────────────────────────────────────────────────────
   - Aucun feedback visuel après création/modification/suppression
   - INotificationService existe mais implémentation Mock sans UI
   - Besoin : système de toast/notification global pour les 
     actions CRUD (ex: "Ressource créée avec succès")

─────────────────────────────────────────────────────────
13. PAGE DE STATISTIQUES PUBLIQUES
─────────────────────────────────────────────────────────
   - Backend : StatsController (à vérifier)
   - Gateway : /api/stats/{everything}
   - Aucune UI frontend

─────────────────────────────────────────────────────────
14. GESTION DES PARAMÈTRES ADMIN
─────────────────────────────────────────────────────────
   - Page admin pour la configuration (rôles, permissions, etc.)
   - Backend : AdminController (GetRoles, CreateRole, etc.)
   - Gateway : /api/v1/admin/{everything}
   - Partiellement dans le dashboard admin

─────────────────────────────────────────────────────────
15. AUTH — PAGE DE DEMANDE DE RÉINITIALISATION MOT DE PASSE
─────────────────────────────────────────────────────────
   - ForgotPasswordAsync existe dans AuthService
   - Page de formulaire "Mot de passe oublié ?" manquante
   - Backend prêt : POST /api/auth/forgot-password

─────────────────────────────────────────────────────────
16. TOURS DE PAGE / SKELETON LOADING
─────────────────────────────────────────────────────────
   - La plupart des pages utilisent des spinners simples
   - Pas de skeleton loading pour améliorer la perception de performance

─────────────────────────────────────────────────────────
LÉGENDE
─────────────────────────────────────────────────────────
  [BACKEND PRÊT]  = Le backend et la gateway sont opérationnels
  [BACKEND PARTIEL] = Certains endpoints existent, d'autres non
  [UI MANQUANTE]   = Aucune interface frontend
  [UI INCOMPLÈTE]  = Interface partielle à améliorer
